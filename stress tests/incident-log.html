Pflugerville parents couldn't see star charts in the iClass app. Kept getting "unavailable" when entering the code from the email sent to families.

The "Star Chart update — what's changing April 6th" email sent from CPF's ActiveCampaign listed the Organization Name as capgympflug. The correct code (confirmed via app.iclasspro.com/a/capgymhp) is capgymhp. Parents were typing a code that doesn't resolve to any iClass org — hence "unavailable."

Any email to parents/families that mentions an org code must have the code copy-pasted from the gym's actual Customer Portal URL — never typed, never paraphrased. Cross-check against the Help & Updates panel's mobile app card before sending.

Critical Vercel Deploy Failure — All PDF Generation Down April 5, 2026
What the manager saw
Uploaded XLS file, clicked "Convert Eval Scores." Got this error instead of a PDF:
PDF generation failed: Unexpected token 'A', "A server e"... is not valid JSON
Impact
100% of PDF generation blocked. No manager at any gym could generate star charts. The tool appeared completely broken. Locally (localhost:5050) everything worked fine — the issue was only on the live Vercel deployment.
Vercel Error Logs
POST /generate-pdf → 500

could not import "api/generate-pdf.py":
Traceback (most recent call last):
  File "/var/task/api/generate-pdf.py", line 7, in <module>
    from pdf_generator import generate_pdf, generate_multi_pdf
ModuleNotFoundError: No module named 'pdf_generator'
Error: Bundle size (458.66 MB) exceeds Lambda limit (245 MB)
Root Cause 1 — Python Import Path Broke
Vercel updated their Python runtime. The import statement from pdf_generator import ... in api/generate-pdf.py used to work because Vercel automatically added the api/ directory to Python's module search path. After their update, they stopped doing that. The file pdf_generator.py was right next to generate-pdf.py in the same folder — Python just couldn't find it anymore.
Root Cause 2 — Bundle Size Exceeded Limit
Files accumulated in the repo over weeks of development: node_modules/ (SheetJS), stress tests/, CLASS-RENAME-TOOL/, layout-attempts/, auditing tools/. Total: 458MB. Vercel's Lambda limit: 250MB. No .vercelignore file existed to tell Vercel which folders to skip.
Timeline
Earlier that day
Multiple commits pushed (walkthrough fixes, format guide updates). Vercel auto-deployed each one.
~11:19 AM CT
First failed request logged. POST /generate-pdf returns 500. Manager tries to generate PDF from real HGA data for the first time.
~11:21 AM CT
Second failed request. Confirmed: every request fails. Error: ModuleNotFoundError.
~11:25 AM CT
Root cause identified from Vercel runtime logs. Import path broken + bundle oversized.
~11:30 AM CT
Fix deployed: sys.path.insert added to generate-pdf.py. .vercelignore created. Commit 690db5e pushed.
~11:32 AM CT
Vercel rebuild triggered. Waiting for deploy to complete.
Fix Applied
Fix 1 — Import path: Added two lines to api/generate-pdf.py before the import:
import os, sys
sys.path.insert(0, os.path.dirname(os.path.abspath(__file__)))
This tells Python exactly where to find pdf_generator.py regardless of what Vercel does with its runtime.

Fix 2 — Bundle size: Created .vercelignore in the repo root listing every large folder that doesn't need to be in the serverless function: node_modules, stress tests, CLASS-RENAME-TOOL, layout-attempts, auditing tools, reference-pdfs, .playwright-mcp.

Commit: 690db5e
Prevention Rules (Added Permanently)
1. After EVERY git push, check Vercel logs within 2 minutes for errors
2. Test evalconvert.com after every deploy — localhost working does NOT mean production works
3. When adding new folders to the repo, add them to .vercelignore immediately
4. Never remove the sys.path.insert line from api/generate-pdf.py
5. If Vercel updates their Python version, check for import errors in the logs
